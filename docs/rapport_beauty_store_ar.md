# تقرير مشروع Beauty Store

## 1. المقدمة

Beauty Store هو موقع تجارة إلكترونية مخصص لبيع منتجات التجميل والعناية بالبشرة. الموقع يسمح للزبون بتصفح المنتجات، إضافة المنتجات إلى السلة، إنشاء طلب، ثم متابعة حالة الطلب. كما يحتوي على لوحة تحكم خاصة بالمدير لإدارة المنتجات، الطلبات، المستخدمين، والمدفوعات.

الفكرة الأساسية من المشروع هي إنشاء متجر بسيط وحديث يناسب الاستخدام المحلي. لذلك تم اعتماد طريقة دفع Bankily B-pay، مع مراجعة يدوية من طرف المدير بدل الاعتماد على Stripe أو بطاقة بنكية.

## 2. هدف المشروع

هدف المشروع هو بناء منصة كاملة لبيع منتجات التجميل عبر الإنترنت.

الأهداف الرئيسية:

- عرض المنتجات بطريقة واضحة وجميلة.
- السماح للزائر باستعمال السلة بدون تسجيل.
- طلب تسجيل الدخول فقط عند الدفع.
- إنشاء طلب من محتوى السلة.
- تسجيل كود Bankily B-pay.
- تمكين المدير من قبول أو رفض الدفع.
- تمكين الزبون من متابعة طلباته.
- تمكين المدير من إدارة المنتجات، التصنيفات، المستخدمين، الطلبات والمدفوعات.

## 3. التقنيات المستخدمة

المشروع مقسم إلى backend و frontend.

### Backend

الـ backend مبني باستخدام FastAPI، وهو مسؤول عن منطق السيرفر.

التقنيات:

- Python
- FastAPI
- SQLAlchemy
- MySQL
- Alembic
- JWT
- bcrypt

### Frontend

الـ frontend مبني باستخدام React و JavaScript.

التقنيات:

- React
- Vite
- JavaScript
- CSS
- lucide-react للأيقونات
- Playwright للاختبارات

## 4. بنية المشروع

### Backend

أهم المجلدات:

- `app/models`: جداول قاعدة البيانات.
- `app/schemas`: التحقق من البيانات.
- `app/routes`: API routes.
- `app/utils`: أدوات مساعدة مثل auth و validators.
- `uploads/products`: صور المنتجات.
- `alembic`: migrations.
- `data/seed_demo_data.py`: إدخال المنتجات والبيانات الأولية.

### Frontend

تم تقسيم الواجهة إلى:

- `website`: واجهة الزبون.
- `admin`: لوحة تحكم المدير.
- `shared`: ملفات مشتركة مثل API client و context و helpers.

هذا التقسيم يجعل المشروع أوضح وأسهل في التطوير.

## 5. منطق الزبون

### الصفحة الرئيسية

تعرض الصفحة الرئيسية اسم المتجر، صورة رئيسية، زر Shop Now، وقسم منتجات مميزة. المنتجات تأتي من قاعدة البيانات. وإذا لم يعمل backend، توجد بيانات احتياطية حتى لا تظهر الصفحة فارغة.

### صفحة المنتجات

تعرض كل المنتجات الموجودة في قاعدة البيانات. كل منتج يظهر معه:

- الصورة
- الاسم
- السعر

صور المنتجات مخزنة في backend داخل `uploads/products`.

### صفحة تفاصيل المنتج

تعرض:

- اسم المنتج
- السعر
- الوصف
- الصورة
- المخزون
- اختيار الكمية
- زر Add to Cart

يمكن للزائر إضافة منتج للسلة بدون تسجيل الدخول.

### السلة

السلة تعمل للزائر وللمستخدم المسجل.

إذا الزائر غير مسجل، يتم حفظ السلة محليًا في المتصفح. عند تسجيل الدخول، يمكن مزامنة السلة مع backend.

الزبون يستطيع:

- تغيير الكمية
- حذف منتج
- رؤية المجموع
- الذهاب إلى الدفع

### التسجيل وتسجيل الدخول

لا يجبر الموقع الزبون على التسجيل من البداية. التسجيل مطلوب فقط عند checkout. هذا يجعل تجربة المستخدم أفضل لأن الزبون يستطيع مشاهدة المنتجات أولًا بدون عائق.

كلمات المرور يتم حفظها بشكل آمن باستخدام bcrypt.

## 6. منطق الدفع Bankily B-pay

الدفع يدوي وليس تلقائيًا.

العملية:

1. الزبون يضيف المنتجات إلى السلة.
2. يذهب إلى checkout.
3. إذا لم يكن مسجلًا، يطلب منه تسجيل الدخول.
4. يرسل المال عبر Bankily B-pay.
5. يدخل كود B-pay المكون من 4 أرقام.
6. backend ينشئ order و payment.
7. payment تصبح حالتها `under_review`.
8. المدير يراجع الدفع.
9. إذا كان صحيحًا يقبله.
10. إذا كان غير صحيح يرفضه.

هذه الطريقة مناسبة الآن لأن الدفع التلقائي عبر Bankily غير مدمج.

## 7. لوحة تحكم المدير

لوحة التحكم منفصلة عن واجهة الزبون. ولا يدخلها إلا المستخدم الذي دوره `admin`.

### Dashboard

يعرض:

- عدد الطلبات
- الإيرادات
- العملاء
- الطلبات الملغاة
- الرسوم البيانية
- أحدث الطلبات

### إدارة المنتجات

المدير يستطيع:

- إضافة منتج
- تعديل منتج
- حذف منتج
- تغيير السعر
- تغيير المخزون
- رفع صورة منتج

عند رفع صورة، تحفظ في:

`backend/uploads/products`

وقاعدة البيانات تخزن رابط الصورة فقط.

### إدارة الطلبات

المدير يستطيع رؤية كل الطلبات. زر View يفتح تفاصيل الطلب:

- الزبون
- السعر الإجمالي
- الحالة
- العنوان
- المنتجات المطلوبة

كما يمكنه تغيير حالة الطلب حسب المنطق:

- من `pending` إلى `cancelled`
- من `paid` إلى `shipped`
- من `shipped` إلى `delivered`

### إدارة المدفوعات

صفحة Payments تعرض مدفوعات Bankily التي تنتظر المراجعة.

المدير يستطيع:

- Approve
- Reject

إذا قبل الدفع، يصبح الطلب paid. وإذا رفضه، تصبح حالة الدفع rejected.

### إدارة المستخدمين

يمكن للمدير رؤية المستخدمين وتغيير دورهم بين customer و admin.

## 8. قاعدة البيانات

قاعدة البيانات الأساسية هي MySQL.

أهم الجداول:

- users
- products
- categories
- cart_items
- orders
- order_items
- payments

كل منتج ينتمي إلى تصنيف. كل طلب ينتمي إلى مستخدم. كل طلب يحتوي على عناصر طلب. وكل طلب يمكن أن يكون له payment واحد.

يتم استخدام Alembic لإدارة تغييرات قاعدة البيانات بدل إنشاء الجداول يدويًا.

## 9. الأمان

المشروع يحتوي على:

- تشفير كلمات المرور باستخدام bcrypt.
- JWT authentication.
- حماية routes الخاصة بالمدير.
- validation للبيانات.
- rate limit بسيط على login/register.
- logout يلغي التوكن في الذاكرة.
- تحديد نوع وحجم الصور المرفوعة.

في production الأفضل استخدام Redis لحفظ revoked tokens و rate limits.

## 10. تخزين الصور

صور المنتجات موجودة في:

`backend/uploads/products`

وقاعدة البيانات تخزن فقط المسار:

`/uploads/products/product-name.png`

الـ frontend يأخذ هذا المسار ويعرض الصورة من backend.

هذه طريقة جيدة كبداية. لكن لاحقًا، إذا صار المشروع حقيقيًا وكبيرًا، الأفضل استخدام Cloudinary أو Amazon S3 أو DigitalOcean Spaces.

## 11. الاختبارات

يوجد اختبارات backend و frontend.

اختبارات backend تستخدم SQLite في الذاكرة حتى لا تلمس قاعدة البيانات الرئيسية.

اختبارات frontend تستخدم Playwright لاختبار الموقع في المتصفح، مثل flow الخاص بالشراء والدفع.

## 12. تشغيل المشروع

يوجد ملف:

`start-dev.ps1`

يساعد على تشغيل backend و frontend معًا أثناء التطوير.

backend يعمل على:

`http://localhost:8000`

frontend يعمل على:

`http://localhost:5173`

## 13. ما تبقى للتطوير

المشروع جيد ويعمل، لكن ما زال يمكن تطوير:

- الدفع التلقائي إذا توفرت API من Bankily.
- اختبارات أكثر.
- تجهيز production deploy.
- backup للصور.
- تحسين dashboard charts.
- تحسين رسائل الأخطاء.
- إضافة Docker لاحقًا.

## 14. الخلاصة

Beauty Store هو مشروع e-commerce متكامل لبيع منتجات التجميل. يحتوي على واجهة زبون، سلة، طلبات، دفع Bankily B-pay، ولوحة تحكم للمدير.

المشروع منظم بشكل جيد وقابل للتطوير. أهم ما تبقى هو تحسين الاختبارات، تجهيز النشر، وتحسين بعض أجزاء لوحة التحكم.
